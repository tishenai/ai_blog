# 旅行记账系统 - 使用指南

## 🎯 功能概述

这是一套基于飞书多维表格的旅行记账系统，支持：

- **问卷式录入**：不用直接操作表格，通过交互卡片填写消费
- **智能分摊**：支持3人平摊、2人平摊、个人付款等各种场景
- **一键统计**：查看总消费、每人消费、净余额
- **最优转账**：贪心算法计算最少转账次数结清所有账目
- **模板复用**：可分享模板，快速创建新的旅行记账本

## 🖐️ 交互命令

### 1. 添加消费记录

发送以下格式的消息来添加消费：

```
/add 消费项目 [消费类型] [金额] [付款人] [平摊人数] [平摊成员...]
```

**示例**:

```
/add 晚餐 餐饮 300 小明 3 小明,小红,小华
/add 机票 交通 900 小明 3 小明,小红,小华
/add 酒店 住宿 600 小红 3 小明,小红,小华
/add 门票 门票 150 小红 2 小明,小华
/add 购物 购物 200 小明 1 小明
```

### 2. 查看统计

```
/stats
```

显示：总消费、每人已付/应摊/差额、净余额、最优转账方案

### 3. 一键平账

```
/settle
```

显示最优转账方案，说明谁该给谁多少钱

### 4. 添加/删除成员

```
/addmember 小明
/delmember 小明
```

### 5. 标记结算

```
/paid [消费ID]
```

将某笔消费标记为已结算

## 🔧 技术实现

### 贪心算法说明

算法计算最优转账方案的逻辑：

1. 计算每个人的净余额（已付 - 应摊）
2. 正余额=应收，负余额=应付
3. 每次让最大债务人向最大债权人转账
4. 重复直到所有人的余额归零

**优势**：最少转账次数，通常比简单平均减少约50%的转账数

### 平摊计算逻辑

- **3人平摊**：每人付 1/3
- **2人平摊**：每人付 1/2
- **1人付款**：付款人全额，其他人欠他
- **跨成员平摊**：按实际参与人数计算
